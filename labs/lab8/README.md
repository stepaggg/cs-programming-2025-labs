# Финальная лабораторная работа

Финальная лабораторная работа состоит из выполнения задания по техническому заданию (ТЗ), оформления отчета по СТО и очной защиты выполненной работы (код и отчет)

Варианты работы:

* RPG текстовая игра [(Тех. Задание)](final/rpg-game.md)
* Система управления заправочной станцией [(Тех. Задание)](final/gas-station.md)

Вариант работы выбирается студентом самостоятельно

## Критерии оценивания

1. Соответствие требованиям из ТЗ
2. Правильная и стабильная работа программы (отсутствие аварийных завершений, корректная обработка ошибок и некорректных входных данных)
3. Чистота и понятность кода (логичная структурированность, использование функций, наличие комментариев)
4. Оформление отчета по СТО
5. Содержание отчета с полным и понятным описанием работы программы (пояснены все основные сложные функции, описана общая структура работы программы)
6. Очная защита работы (ответы студента на вопросы по программе, отчету и исходному коду)
7. Наличие опубликованного исходного кода в учебном git репозитории (включая цифровую версию отчета, docx или pdf)

При наличии замечаний по любому из критериев итоговый балл за работу может быть снижен

## Защита работы

Отчет должен быть сдан __в печатном виде__:

* преподавателю
* или на кафедру (1448)

Допускается двухсторонняя печать ЧБ или цветная

Если в отчете допущены ошибки, связанные с СТО, или имеются проблемы с содержанием - отчет не принимается и студент не допускается к защите финальной работы. Отчет можно переделать и попробовать сдать повторно

После сдачи отчета студент допускается к защите финальной работы. На защите будет проверена работоспособность программы, а также будут заданы вопросы студенту по коду программы, отчету и работе самой программы

Код программы и цифровая версия отчета (docx или pdf) должны быть опубликованы в git репозитории

Работа считается сданной при наличии подписи преподавателя и выставленного балла за работу
